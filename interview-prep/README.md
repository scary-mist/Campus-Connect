# Campus Connect — Interview Preparation Kit

A comprehensive guide for discussing the Campus Connect project in technical interviews.

## 📁 Files

| File | Purpose |
|------|---------|
| [PROJECT_OVERVIEW.md](PROJECT_OVERVIEW.md) | 30-second and 2-minute pitch, tech stack rationale |
| [ARCHITECTURE.md](ARCHITECTURE.md) | MVVM, repos, Hilt, data flows (messaging, image upload, wave states) |
| [TECHNICAL_DEEP_DIVE.md](TECHNICAL_DEEP_DIVE.md) | Geohashing, transactions, callbackFlow, friend requests, image uploads, reactive nav |
| [INTERVIEW_QA.md](INTERVIEW_QA.md) | 36 Q&A covering arch, Firebase, proximity, chat, security, scaling, pull-to-refresh |
| [RESUME_BULLET_POINTS.md](RESUME_BULLET_POINTS.md) | Concise, detailed, and system-design-focused resume options |
| [SCALABILITY.md](SCALABILITY.md) | Bottleneck analysis and scaling strategy for 100K+ users |
| [CHALLENGES_FACED.md](CHALLENGES_FACED.md) | 11 real challenges faced during development with root causes and solutions |
| [BUILD_FROM_SCRATCH_GUIDE.md](BUILD_FROM_SCRATCH_GUIDE.md) | Step-by-step flowchart to build the app from zero, with curated docs + video resources |

## 🎯 How to Use

1. **Before interviews**: Read PROJECT_OVERVIEW for your pitch
2. **For architecture rounds**: Study ARCHITECTURE + TECHNICAL_DEEP_DIVE
3. **For behavioral rounds**: Use CHALLENGES_FACED as your story bank
4. **For system design**: Reference SCALABILITY
5. **For resume writing**: Pick from RESUME_BULLET_POINTS
